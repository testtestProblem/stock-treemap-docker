1. 🔍 自選清單動態搜尋與新增 (Watchlist Dynamic Input)
顯示條件 (Conditional Rendering)：

當 Tabs 狀態切換為 自選清單 1 或 自選清單 2 時，左側列表的最下方必須動態渲染一個 Input Bar。若當前切換在 [庫存] Tab，則隱藏該輸入框。

輸入框行為 (Input Behavior)：

支援股號數字（如 5443）或股名/廠商名稱（如 均豪）的模糊搜尋（Fuzzy Search）。

點擊右側 [+] 按鈕或按下 Enter 鍵時，觸發 API 查詢，並在驗證成功後將該檔股票格式化加入至當前的自選清單陣列（Array）末端。

2. ↕️ 列表/欄位拖拽排序 (Drag-and-Drop Reordering)
拖拽機制 (UI UX)：

在自選清單的每檔股票卡片左側，新增一個拖拽手柄圖示（如 ☰），允許使用者透過滑鼠或觸控上下拉動改變個股的垂直排列順序。


3. 💾 順序持久化記憶 (State Persistence)
記憶邏輯 (Data Binding)：

當拖放行為結束（onDragEnd）時，系統需立即捕捉新的排序陣列索引（Updated Array Indices）。

本地緩存（快速響應）：將變更後的順序暫存於前端 中。

後端同步（永久記憶）：更新資料庫中該使用者 profile 的自選清單排序參數，確保跨裝置登入或重新整理網頁時，自選排序與欄位配置能完美復原。
